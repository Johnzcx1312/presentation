# talent-search-strategy

## 何时使用
当用户提出招聘搜索诉求、需要从历史上下文恢复筛选条件、并希望自动改写搜索条件提升结果质量时，使用本 skill。

## 核心流程
1. 调用 `resume_search_workflow`。
2. 将输入拆分为：
   - `fixed_filters`（城市/经验/学历/年龄/性别/语言等固定字段）
   - `keyword_plan`（职位、公司、技能、业务方向等关键词维度）
3. 执行查询并评估质量（too_many / too_few / drift / mismatch）。
4. 必要时自动改写并重试，最多 3 轮。
5. 返回可用结果；若未达标返回 best effort + 缺失信息建议。

## 用户说明规范
- 对用户输出简洁：结果概览 + 1-2 句诊断建议。
- 不暴露内部 prompt 原文，不输出无法解析格式。
- 若结果不理想，明确建议补充信息（如目标公司、必须技能、职级范围）。
